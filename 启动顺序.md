# 项目启动顺序（后端 + 前端）

## 0. 环境要求
- JDK 17
- Maven 3.6+
- Node.js 18+（当前项目已在 Node 22 验证可用）
- MySQL 8.x（本地默认 `127.0.0.1:3306`）
- 可选：Redis 6379、RabbitMQ 5672（本项目已做本地开发兜底，不启动也可进入页面）

## 1. 初始化数据库
在项目根目录 `E:\01XM\demo\20260314` 执行：

```powershell
mysql -uroot -p1234 -e "CREATE DATABASE IF NOT EXISTS mall_demo DEFAULT CHARSET utf8mb4;"
mysql -uroot -p1234 mall_demo < src\main\resources\sql\user_account.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\product.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\product_description_migration.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\product_detail.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\order.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\seckill_product.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\seckill_order.sql
mysql -uroot -p1234 mall_demo < src\main\resources\sql\init_data.sql
```

## 2. 启动后端（Spring Boot）
在项目根目录执行：

```powershell
mvn -DskipTests spring-boot:run
```

后端地址：
- `http://localhost:8080`

## 3. 启动前端（Vite）
新开一个终端，进入前端目录执行：

```powershell
cd frontend
npm install
npm run dev
```

前端地址：
- `http://localhost:5173`

## 6. 常见问题
### 6.1 后端启动报数据库连接错误
- 检查 MySQL 是否已启动。
- 检查数据库账号密码是否和 `src/main/resources/application.yml` 一致。

### 6.2 前端能开但接口 404
- 确认后端已启动在 `8080`。
- 确认前端 `.env.development` 中 `VITE_API_TARGET=http://localhost:8080`。

### 6.3 页面空数据
- 重新执行 `src/main/resources/sql/init_data.sql` 导入演示数据。
