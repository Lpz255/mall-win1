# 前后端联调手册（电商系统）

## 1. 联调前检查
1. 后端服务已启动：`http://localhost:8080`
2. 前端开发服务已启动：`http://localhost:5173`
3. `vite` 代理配置正确：
   - `VITE_API_TARGET=http://localhost:8080`
   - `VITE_PROXY_STRIP_PREFIX=true/false` 与后端路由保持一致
4. 后端统一返回结构：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

## 2. 本次联调修复点
1. 请求参数适配：统一时间格式为 `yyyy-MM-dd HH:mm:ss`，ID 入参自动规范化。  
2. 异常码处理：统一 `400/401/403/404/409/429/500` 中文提示。  
3. 中文乱码修复：后端消息出现编码异常时，前端自动尝试修复再展示。  
4. 鉴权分流：`/admin/*` 使用 `admin_token`，`/web/*` 使用 `token`。  
5. 高频缓存：商品列表/分类/热门数据增加本地缓存（TTL）。  

## 3. 常见问题与解决方案
### 3.1 跨域失败（404 或 CORS）
现象：
- 浏览器控制台提示 CORS 报错
- `/api/xxx` 返回 404

排查与处理：
1. 检查 `vite.config.js` 里的 `proxy.target` 是否正确。
2. 检查 `VITE_PROXY_STRIP_PREFIX`：
   - 后端接口是 `/product/list`：设为 `true`
   - 后端接口是 `/api/product/list`：设为 `false`
3. 重启前端服务使 `.env` 生效。

### 3.2 登录后仍跳登录页
现象：
- 请求返回 `401`
- 路由守卫重复跳转登录

排查与处理：
1. 用户端确认 `localStorage.token` 是否存在。
2. 后台端确认 `localStorage.admin_token` 是否存在。
3. 检查后端返回 Token 字段名是否在前端兼容列表中（`token/accessToken/jwt`）。
4. 检查接口是否按约定携带 `Authorization: Bearer <token>`。

### 3.3 中文提示乱码
现象：
- 弹窗显示为 `æµè¯` 等乱码字符。

排查与处理：
1. 后端接口响应头应为 `Content-Type: application/json; charset=UTF-8`。
2. Nginx `charset utf-8;` 保持开启。
3. 前端已做乱码兜底修复，如仍异常，优先修正后端编码链路。

### 3.4 参数类型不匹配
现象：
- 后端提示 `productId` 类型错误，或时间格式无法解析。

排查与处理：
1. 前端已自动规范化 ID 和时间字段。
2. 若后端字段命名特殊（非 `id/*Id`），在接口层手动映射参数名。
3. 提交前可在浏览器网络面板确认最终请求体。

### 3.5 商品数据更新后列表未刷新
现象：
- 后台已修改商品，前台列表短时间仍显示旧数据。

说明：
- 前台商品列表启用了短期缓存（默认 30s）。
- 后台商品新增/编辑/删除/上下架后已自动清缓存。

手动处理：
- 刷新页面或等待缓存过期。

## 4. 性能优化策略说明
1. 商品列表缓存：减少重复查询请求。  
2. 倒计时渲染：使用 `requestAnimationFrame` 驱动，减少定时器抖动。  
3. 操作防抖/节流：
   - 购物车数量变更：防抖提交
   - 秒杀按钮：节流 + 提交中禁用
4. 打包拆分：
   - `vue`、`element-plus`、`framework`、`xlsx` 按 chunk 分离
   - 静态资源长期缓存

## 5. 验收建议
1. 用户端：登录 -> 商品列表 -> 详情 -> 加购 -> 下单 -> 秒杀。  
2. 后台端：登录 -> 商品管理增删改 -> 秒杀管理启动停止 -> 订单改状态 -> 用户禁用。  
3. 观察所有弹窗中文是否正常，网络失败场景提示是否友好。  
4. 在浏览器网络面板检查请求次数是否因缓存与防抖下降。
